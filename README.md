# hookah-db

Репозиторий PostgreSQL-схемы проекта Hookah.

## Что хранится здесь

- SQL-миграции
- сиды для локальной разработки
- Docker-образ базы
- скрипт повторного применения миграций для `docker compose`

## Домены

- `auth` — пользователи, логин, пароль-хэш, роли, апрув;
- `catalog` — бренды, линейки и вкусы табака;
- `equipment` — чашки, кальяны, калауды и уголь;
- `recipes` — забивки и состав смесей;
- `inventory` — партии и движения по складу;
- `sales` — заказы по столам, участники, статусы, фактические забивки и отзывы;
- `support` — enum-типы и служебные SQL-функции.

## Локальный запуск

```bash
cp .env.example .env
docker compose up -d
```

При первом старте Postgres применяет `migrations` и `seeds` через init-скрипты.
В общем `compose` дополнительно есть сервис `db-migrate`, который повторно прогоняет миграции и сиды на каждом запуске стека. Это позволяет обновлять схему на уже существующем volume без ручного `down -v`.

## Docker

```bash
docker build -t hookah-db .
docker run --rm -p 5432:5432 --env-file .env hookah-db
```
