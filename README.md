# hookah-db

Репозиторий схемы PostgreSQL для системы управления кальянной. Здесь живут миграции, сиды и локальная инфраструктура базы.

## Текущие домены

- `auth` — пользователи, логин, пароль-хэш, роль, optional email и telegram username.
- `catalog` — бренды табака, линейки и вкусы.
- `equipment` — производители оборудования, чашки, кальяны, калауды и уголь.
- `recipes` — забивки, связанное оборудование и состав смеси по процентам.
- `inventory` — партии и движения по складу.
- `sales` — заказы и позиции.
- `support` — enum-типы, служебные функции и общие ограничения.

## Что уже смоделировано

- Пользователь хранится с `login`, `password_hash` и полем `role`, а `email` и `telegram_username` сделаны необязательными.
- Крепость линейки хранится на уровне `catalog.product_lines`.
- У конкретного вкуса табака есть `estimated_strength_level`, `brightness_level` и `flavor_description`.
- Чашки, кальяны, калауды и уголь вынесены в отдельные таблицы оборудования.
- Забивка хранит связанное оборудование, параметры прогрева, количество угля, общий вес табака и текстовый отзыв.
- Состав смеси вынесен в `recipes.packing_tobaccos`, где на уровне БД проверяется сумма процентов ровно в `100.00`.

## Файлы

- `compose.yaml` — локальный PostgreSQL-контур.
- `migrations` — версия схемы и структура БД.
- `seeds` — демо-данные для локальной разработки.

## Как запускать локально

```bash
cp .env.example .env
docker compose up -d
```

При первом старте контейнер автоматически применит SQL-файлы из `migrations` и `seeds`, включая demo-аккаунты `admin/admin`, `master/master` и `client/client`.

## Docker

Собрать и запустить только PostgreSQL-контейнер этого репозитория:

```bash
docker build -t hookah-db .
docker run --rm -p 5432:5432 --env-file .env hookah-db
```

Для совместного запуска с frontend и backend используйте корневой `compose.yaml` в главном репозитории.
